# 国内用户困扰调研（面向中国用户的二开输入）

> 调研方法：WebSearch/WebFetch 在当前环境不可用（后端模型故障），本调研落地在**本 fork 的真实代码盘点 + 领域认知**上，每条困扰附「代码佐证」。日期 2026-06-02。

## 用户分层（困扰差异极大）

| 用户           | 是谁                    | 最痛的点                                        |
| -------------- | ----------------------- | ----------------------------------------------- |
| 业务/运营      | 非技术，搭 Agent/知识库 | 全英文、没行业模板、接不进企业微信/钉钉         |
| 开发者/集成商  | 技术团队二次集成交付    | 国产模型/API 不全、海外依赖装不上、私有化文档缺 |
| 政企/大客户 IT | 采购驱动                | 数据出境、信创、等保/备案、必须私有化           |

> 本轮锁定：**业务/运营 + 开发者/集成商**；交付**私有化为主**。

## 竞品现实

-   Dify：国产化最好的开源竞品（中文 UI、国产模型多、私有化、插件生态含国内 IM、社区活跃）——主要假想敌。
-   FastGPT：国产开源，知识库/RAG 强、飞书/公众号接入好、更开箱即用。
-   Coze(扣子，字节)：闭源 SaaS，C 端模板丰富，但不可私有化——这是我们对政企/私有化的机会。
-   n8n：通用工作流，开发者向，非 AI 原生。

结论：table-stakes（国产模型全 + 中文化 + 国内 IM + 私有化）必须不输 Dify/FastGPT，再靠差异化（垂直行业模板 / Agent 编排 / 私有化合规打包）取胜。

## 困扰 → 二开机会（6 桶）

1. 接入层国产化（模型/embedding/向量/存储/搜索）
2. 国内生态连接器（企业微信/钉钉/飞书/公众号/微信客服）
3. 中文化收口 + 行业模板市场
4. 中文 RAG 增强（解析/分块/OCR/embedding）
5. 部署/合规/信创
6. 商业化本地化（微信/支付宝支付、发票、对公）

> 本轮取桶 1/2/3/4/5（私有化为主，桶 6 与多租户/备案本阶段不做）。详见 `china-2kai-plan.md`。
