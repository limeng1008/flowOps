## FlowOps 是什么 · 为什么选它

FlowOps 是基于 Flowise 深度二次开发的中文 AI 智能体搭建平台。它保留了可视化编排、节点化调试、知识库问答、工具调用等能力，同时面向中国企业补齐了国产大模型、国产向量模型、全中文界面、办公文档导出、企微/飞书转人工、订单/物流/售后查询、私有化部署与信创环境适配。

**它解决什么痛点**

很多企业不是缺少模型，而是缺少把模型稳定接入业务的“中间层”：业务同事需要能看懂、能试跑；开发同事需要能接凭证、接知识库、接内网 API；管理层关心数据是否出域、能否私有化、是否适合国内供应商生态。FlowOps 把这些能力放到同一个可视化工作台里，让“从模板到上线”少走很多弯路。

**核心价值**

1. **私有化部署**：可按 [deploy/README.md](../../deploy/README.md) 使用 Docker 从源码构建，文件、数据库、凭证密钥默认保存在自有环境。
2. **国产模型优先**：内置 `智谱 GLM`、`Kimi (月之暗面)`、`豆包 (火山方舟)`、`通义千问 (Qwen)`、`MiniMax` 聊天模型节点，以及 `智谱文本向量模型`、`通义文本向量模型`、`硅基流动文本向量模型`。
3. **全中文体验**：默认语言为 `zh`，语言入口支持 `中文` 与 `English`。
4. **业务交付更完整**：不仅能回答问题，还能通过 `文档导出`、`表格导出`、`PPT 导出` 生成 Word、Excel、PPTX 下载链接。
5. **客服场景可闭环**：`转接人工坐席（企业微信/飞书）` 可把用户与客服双方完整对话历史推送到坐席群；`客服查询（订单/物流/售后）` 支持演示数据与自有 API 查询。
6. **界面更清爽**：部署配置里默认隐藏 47 个已弃用或被国产节点替代的上游节点，降低业务用户选错节点的概率。

**操作步骤**

1. 用私有化部署方式启动 FlowOps，首次访问 `/signin` 注册管理员账号。
2. 进入左侧导航，先浏览 `应用市场`、`文档库`、`API 密钥`、`变量`、`工具` 等入口。
3. 从行业模板开始搭建，先跑通一个模板，再按业务替换模型、知识库或工具节点。

## 5 分钟上手

**它解决什么痛点**

新团队最怕一上来就从空白画布开始。FlowOps 的推荐路径是：先用模板跑通业务闭环，再把模型、凭证、知识库、工具替换成自己的配置。这样业务同事可以先看到结果，开发同事再逐步接入真实系统。

**操作步骤**

1. 打开 `应用市场`，选择一个中文模板。客服场景推荐先用 `转人工客服智能体`，办公场景推荐先用 `Excel 表格生成智能体`。
2. 点击使用模板进入画布。模板会保留节点、连线、系统提示词和必要的表单字段。
3. 打开模型节点，把默认模型凭证换成自己的凭证。例如 `智谱 GLM` 使用 `智谱 AI API`，默认模型可先选 `glm-4.5`。
4. 需要知识库时，先到 `文档库` 上传并 Upsert，再回到智能体节点的 `Knowledge (Document Stores)` 里引用。
5. 点击运行。若模板包含 `表格导出`、`文档导出` 或 `PPT 导出`，运行结果会返回下载链接。

## 核心能力

### 国产大模型与向量模型全家桶

**能力简介**

FlowOps 把常见国产模型做成独立节点，业务用户不需要手动记接口地址；开发者只需要配置对应凭证，即可在画布里选择模型并调参。

**它解决什么痛点**

直接使用 OpenAI 兼容接口时，业务用户容易在 Base URL、模型名、密钥、流式输出等参数上出错。FlowOps 将供应商封装成明确的节点与凭证：聊天模型用于生成、推理和对话，向量模型用于知识库检索，降低接入成本。

**真实节点与凭证**

| 类型     | 节点展示名             | 节点名                 | 默认模型            | 凭证                       |
| -------- | ---------------------- | ---------------------- | ------------------- | -------------------------- |
| 聊天模型 | `智谱 GLM`             | `chatZhipuAI`          | `glm-4.5`           | `智谱 AI API`              |
| 聊天模型 | `Kimi (月之暗面)`      | `chatMoonshot`         | `kimi-k2.6`         | `Kimi API`                 |
| 聊天模型 | `豆包 (火山方舟)`      | `chatDoubao`           | `doubao-pro-32k`    | `火山方舟 API`             |
| 聊天模型 | `通义千问 (Qwen)`      | `chatQwen`             | `qwen-plus`         | `阿里云百炼 DashScope API` |
| 聊天模型 | `MiniMax`              | `chatMinimax`          | `MiniMax-Text-01`   | `MiniMax API`              |
| 向量模型 | `智谱文本向量模型`     | `embeddingZhipu`       | `embedding-3`       | `智谱 AI API`              |
| 向量模型 | `通义文本向量模型`     | `embeddingQwen`        | `text-embedding-v3` | `阿里云百炼 DashScope API` |
| 向量模型 | `硅基流动文本向量模型` | `embeddingSiliconflow` | `BAAI/bge-m3`       | `硅基流动 SiliconFlow API` |

聊天模型节点会从供应商 `/models` 接口尝试拉取可用模型；失败或未配置凭证时使用内置模型列表。常见可选项包括 `Custom Model Name`、`Temperature`、`Streaming`、`Max Tokens`、`Top Probability`、`Frequency Penalty`、`Presence Penalty`、`Timeout`、`Stop Sequence`、`Base Options`。向量模型常见可选项包括 `Custom Model Name`、`Strip New Lines`、`Batch Size`、`Dimensions`、`Timeout`、`Base Options`。

**操作步骤**

1. 在凭证管理里新建对应凭证，例如 `阿里云百炼 DashScope API` 或 `硅基流动 SiliconFlow API`。
2. 在画布里添加模型节点，例如 `通义千问 (Qwen)` 或 `智谱文本向量模型`。
3. 选择凭证，再从 `Model Name` 选择模型。需要使用供应商新模型时，可在 `Custom Model Name` 填入模型 ID 覆盖下拉选择。
4. 对生成类节点，按业务调 `Temperature` 与 `Max Tokens`；对知识库检索，优先保持向量模型稳定，避免同一文档库频繁更换 embedding 模型。

### 可视化搭建智能体

**能力简介**

FlowOps 用画布把模型、记忆、知识库、工具、导出能力串起来。用户可以从模板进入，也可以从空白画布逐个拖节点搭建。

**它解决什么痛点**

传统开发里，一个智能体往往散落在代码、Prompt、API 配置和调试脚本中。画布方式让业务规则、节点输入输出和运行结果都能被看见，方便业务、产品、开发共同评审。

**操作步骤**

1. 新建对话流或智能体流，选择 `Start` 作为入口。
2. 接入模型节点：简单生成任务可用 `LLM`，多工具场景可用 `Agent`。
3. 在 `Messages` 中写系统提示词与用户消息；需要上下文时打开 `Enable Memory`，并选择 `Memory Type`，例如 `All Messages`、`Window Size`、`Conversation Summary`、`Conversation Summary Buffer`。
4. 需要外部能力时，在 `Tools` 中加入工具；需要知识库时配置 `Knowledge (Document Stores)` 或 `Knowledge (Vector Embeddings)`。
5. 点击运行，在右侧对话窗口查看响应、工具调用和导出链接。

![智能体画布与节点配置](images/05-canvas-model-config.png)

> 🏷 **图注**：① `Start` 入口节点　 ② `LLM` 或 `Agent` 节点　 ③ 模型、记忆、工具与知识库配置区　 ④ 右侧运行窗口查看输出

### 企业知识库问答

**能力简介**

FlowOps 的知识库流程是：先在 `文档库` 维护文档，再通过向量模型完成 Upsert，最后在智能体节点里引用知识库回答问题。

**它解决什么痛点**

企业客服、制度问答、售后说明、产品文档经常更新。把资料直接塞进 Prompt 容易超长，也难维护。知识库把文档拆分、向量化、检索与回答分开，适合长期维护。

**操作步骤**

1. 进入 `文档库`，新增知识库并填写名称、描述。
2. 添加文档加载器，上传 FAQ、产品说明、售后政策或内部制度文档。
3. 选择向量模型，例如 `智谱文本向量模型`、`通义文本向量模型` 或 `硅基流动文本向量模型`，执行 Upsert。
4. 回到智能体画布，在 `Agent` 节点配置 `Knowledge (Document Stores)`，选择已 Upsert 的文档库；如果使用已有向量库与 embedding，则配置 `Knowledge (Vector Embeddings)`。
5. 打开 `Enable Memory`，让客服智能体能读取上下文，处理追问和补充信息。

### 一键导出办公文档

**能力简介**

FlowOps 不只生成文本，还能把上游输出导成可下载文件：`文档导出` 生成 Word、Markdown、纯文本；`表格导出` 生成 Excel；`PPT 导出` 生成带主题、版式与图表的 PPTX。

**它解决什么痛点**

很多 AI 工作流最后仍要人工复制到 Word、Excel、PPT。导出节点把“生成内容”变成“交付文件”，适合报告、会议纪要、表格数据、汇报材料等办公场景。

**真实节点与选项**

| 节点       | 关键输入                             | 支持格式或选项                                  |
| ---------- | ------------------------------------ | ----------------------------------------------- |
| `文档导出` | `Content`、`Format`、`File Name`     | `Word (.docx)`、`Markdown (.md)`、`Text (.txt)` |
| `表格导出` | `Content`、`Sheet Name`、`File Name` | 支持对象数组与二维数组，生成 `.xlsx`            |
| `PPT 导出` | `Content`、`主题`、`File Name`       | 主题：`商务蓝`、`科技绿`、`暗夜蓝`、`简约灰`    |

`PPT 导出` 的 `Content` 接收演示文稿 JSON：每页可选 `layout` 为 `cover`、`section`、`bullets`、`twoColumn`、`chart`、`closing`。图表页支持 `{ chart: { type:"bar|pie|line", categories:[], series:[{name,values}] } }`，不写 `layout` 时默认按要点页渲染。

**操作步骤**

1. 让上游 `LLM` 或 `Agent` 输出结构化内容。表格建议输出对象数组或二维数组；PPT 建议输出包含 `slides` 数组的 JSON。
2. 连接对应导出节点。`文档导出` 的 `Content` 引用上游文本；`表格导出` 的 `Content` 引用 JSON 表格；`PPT 导出` 的 `Content` 引用演示文稿 JSON。
3. 设置文件名。PPT 再选择 `主题`，例如 `商务蓝` 或 `科技绿`。
4. 运行后在对话输出中点击下载链接。

### 国产生态连接

**能力简介**

FlowOps 针对客服和运营场景内置两个实用工具：`转接人工坐席（企业微信/飞书）` 与 `客服查询（订单/物流/售后）`。前者把会话推给人工坐席，后者查询订单、物流、退款/售后进度。

**它解决什么痛点**

AI 客服不能只会回答 FAQ。真实场景里，用户会追问订单、物流、退款，也会要求人工。FlowOps 允许智能体在知识库之外调用工具，把“答不上来”变成“转给人”或“查系统”。

**转人工工具**

`转接人工坐席（企业微信/飞书）` 的平台选项是 `企业微信` 与 `飞书`。关键输入包括 `平台`、`消息标题`、`工具名称`、`工具描述（触发时机）`。默认工具名是 `human_handoff`，默认消息标题是 `AI 客服转人工`。凭证为 `群机器人 Webhook（企业微信/飞书）`，字段包括 `Webhook 地址` 与 `飞书签名 Secret（可选）`。

调用成功后，工具会把双方完整对话记录整理为坐席可读文本，并推送到企业微信或飞书群机器人。默认触发场景包括：知识库未覆盖、用户明确要求人工、订单/支付/物流/售后纠纷或投诉。

**客服查询工具**

`客服查询（订单/物流/售后）` 支持两种 `数据来源`：`示例数据（免后端，演示/测试用）` 与 `调用 API（填你自己的接口）`。API 模式下可配置 `订单查询 API`、`物流查询 API`、`退款/售后查询 API`，用 `{value}` 占位订单号或快递单号；可选凭证是 `客服查询 API 鉴权（可选）`，字段是 `Authorization 头的值`。

工具调用参数包括 `queryType` 与 `queryValue`。`queryType` 可为 `order`、`logistics`、`aftersales`，分别代表订单详情/状态、物流快递跟踪、退款/售后进度。

**操作步骤**

1. 客服智能体先配置知识库，让常见问题由知识库回答。
2. 在 `Tools` 里添加 `转接人工坐席（企业微信/飞书）`，选择 `企业微信` 或 `飞书`，并配置 `群机器人 Webhook（企业微信/飞书）` 凭证。
3. 如需查询业务数据，再添加 `客服查询（订单/物流/售后）`。演示阶段选择 `示例数据（免后端，演示/测试用）`；上线前切到 `调用 API（填你自己的接口）`。
4. 在智能体系统提示词里说明：知识库能覆盖就直接回答，超出范围、用户要求人工、涉及订单/支付/物流/售后纠纷或投诉时调用 `human_handoff`。

### 开箱即用行业模板

**能力简介**

FlowOps 在 `packages/server/marketplaces/agentflowsv2/` 中内置 7 个中文行业模板。模板不是泛泛的 Prompt 展示，而是带表单字段、节点、模型配置和说明便签的可运行智能体流。

**它解决什么痛点**

业务用户往往知道自己要“写营销文案”“整理会议纪要”“做 Excel 表格”，但不知道该从哪些节点开始。模板把常见场景做成开箱入口，让用户先跑出结果，再迭代细节。

**模板清单**

### 全中文界面

**能力简介**

FlowOps 的源码常量 `DEFAULT_LANGUAGE` 设置为 `zh`，支持 `中文` 与 `English` 双语切换。国内交付时，业务用户首次进入就是中文界面。

**它解决什么痛点**

上游工具常把大量菜单、弹窗、节点参数、错误提示保留为英文，业务团队难以独立试用。FlowOps 将核心界面、节点说明、弹窗与常见操作补成中文，降低培训成本。

**操作步骤**

1. 首次进入系统时默认使用 `中文`。
2. 右上角语言入口可切换到 `English`，适合开发者对照上游概念。
3. 若页面出现未翻译文本，优先记录页面路径、弹窗或节点位置，再统一补到语言包。

![语言切换](images/13-language-switch.png)

> 🏷 **图注**：① 右上角语言入口　 ② 当前语言 `中文`　 ③ 可切换到 `English` 便于技术人员对照

## 私有化部署

**能力简介**

FlowOps 提供 [deploy/README.md](../../deploy/README.md) 作为私有化部署入口。镜像从本仓库源码构建，包含国产模型、embedding、导出节点、企微/飞书转人工、客服查询与全中文界面。

**它解决什么痛点**

国内企业常要求数据不出域、凭证本地加密、可在内网或信创环境部署。FlowOps 的部署配置默认关闭遥测，文件和凭证密钥本地存储，并可对接 Kingbase。

**操作步骤**

1. 进入 `deploy` 目录，复制 `.env.example` 为 `.env`。
2. 修改数据库密码，以及 `FLOWISE_SECRETKEY_OVERWRITE`、`JWT_AUTH_TOKEN_SECRET`、`JWT_REFRESH_TOKEN_SECRET`、`EXPRESS_SESSION_SECRET`、`TOKEN_HASH_SECRET` 五个密钥。
3. 执行 `docker compose up -d --build`。首次构建约 15-25 分钟。
4. 启动后访问 `http://服务器IP:3000`，首次进入 `/signin` 注册的第一个账号即为管理员。
5. 合规默认项包括 `DISABLE_FLOWISE_TELEMETRY=true`、`STORAGE_TYPE=local`、`SECRETKEY_STORAGE_TYPE=local`。
6. 信创环境可在麒麟(Kylin V10)、统信(UOS) 等 Linux 上安装 Docker 与 Docker Compose 后运行；人大金仓 Kingbase 按 PG 兼容方式配置。
7. 离线 / 内网（airgap）环境可先在有网机器 `docker compose build`，再 `docker save flowops:local postgres:16-alpine -o flowops-images.tar`，到内网 `docker load` 后启动。

![私有化部署命令](images/16-deploy-compose.png)

> 🏷 **图注**：① 复制 `.env.example`　 ② 修改密钥和数据库配置　 ③ 执行 `docker compose up -d --build`　 ④ 首次 `/signin` 注册管理员

## 常见问题 FAQ

**1. 可以直接用官方 `flowiseai/flowise` 镜像吗？**

不建议。私有化部署文档明确说明：官方镜像不包含 FlowOps 的二开内容，包括国产模型、国产 embedding、文档/表格/PPT 导出、企微飞书转人工、客服查询与全中文界面。请使用本仓库源码构建。

**2. 模板默认用哪个模型？**

中文行业模板默认使用 `智谱 GLM`，模型配置为 `chatZhipuAI`，常见默认模型是 `glm-4.5`。导入模板后需要连接自己的 `智谱 AI API` 凭证，也可以替换成其他国产聊天模型节点。

**3. 知识库应该选哪个向量模型？**

可用 `智谱文本向量模型`、`通义文本向量模型`、`硅基流动文本向量模型`。建议同一个文档库长期使用同一种 embedding 模型，避免检索效果因模型切换而波动。

**4. `表格导出` 为什么导出失败？**

`表格导出` 需要对象数组或二维数组。模板里的 `Excel 表格生成智能体` 已经将 `Excel 表格数据生成` 的输出连接到 `表格导出`，若失败，优先检查上游 LLM 输出是否为可解析 JSON。

**5. `PPT 导出` 支持哪些主题和版式？**

主题为 `商务蓝`、`科技绿`、`暗夜蓝`、`简约灰`。版式通过 JSON 每页的 `layout` 指定，支持 `cover`、`section`、`bullets`、`twoColumn`、`chart`、`closing`。

**6. 转人工会把哪些内容推给坐席？**

`转接人工坐席（企业微信/飞书）` 会整理用户与客服双方完整对话历史，并附上转接原因、用户当前问题和 AI 小结，推送到企业微信或飞书群机器人。

**7. 客服查询一定要接真实后端吗？**

不需要。`客服查询（订单/物流/售后）` 的 `数据来源` 可以先选 `示例数据（免后端，演示/测试用）`，用于演示和调试；上线前再切到 `调用 API（填你自己的接口）`。

**8. 数据是否会外发？**

部署配置默认 `DISABLE_FLOWISE_TELEMETRY=true`、`STORAGE_TYPE=local`、`SECRETKEY_STORAGE_TYPE=local`。模型调用是否出域取决于你在节点里配置的模型服务或内网接口地址。
