# FlowOps 上线 Smoke Checklist

日期：2026-06-05
适用分支：`release/flowops-commercialization-v1`
适用场景：封闭内测、私有化交付预验收、正式上线前回归。

## 1. 验收原则

-   每次 release candidate 必须记录 commit、环境、账号、组织、工作区、执行人、时间和结论。
-   每条 smoke 必须留下证据：截图路径、接口日志、服务端日志、导出文件、云厂商控制台记录或订单号。
-   结果只允许三类：`PASS`、`FAIL`、`BLOCKED`。
-   外部凭证缺失、商户未开通、公网回调不可用时记为 `BLOCKED`，不能记为 `PASS`。
-   商业化 V1 默认不开放自助支付；支付 smoke 只验证沙箱/后续开放条件，不作为当前上线入口。

## 3. P0 必跑 Smoke

### S01 登录与权限

目标：确认普通用户、组织管理员、计费/工单运营权限可用。

步骤：

1. 使用普通用户登录。
2. 进入 `/account`，确认能看到当前组织、本地计费套餐和额度。
3. 使用组织管理员登录，确认能进入成员/工作区相关页面。
4. 使用 `BILLING_ADMIN_EMAILS` 白名单账号进入 `/billing`。
5. 使用非白名单账号访问 `/billing`，应返回 403 或无入口。

通过标准：

-   登录成功，无控制台白屏。
-   普通用户不能访问计费后台。
-   白名单运营能看到组织列表、套餐、额度和到期时间。

### S02 创建与运行 Agent 工作流

目标：确认核心 Agent 工作流可创建、保存、运行。

步骤：

1. 新建 AgentflowV2。
2. 配置国产聊天模型，例如 `chatZhipuAI + glm-4.5`。
3. 输入中文业务问题并运行。
4. 打开执行记录，确认输入、输出、状态和耗时可查看。

通过标准：

-   工作流保存成功。
-   模型返回中文结果。
-   执行记录状态为成功。
-   无 500、白屏、节点加载失败。

### S03 应用市场模板导入

目标：确认常用模板能导入画布并跑通。

必测模板：

-   `营销文案生成智能体`
-   `报告生成智能体`
-   `Excel 表格生成智能体`
-   `PPT Deck Agent`
-   任意一家国产云向量库 RAG 模板（有真实向量库凭证时执行）

步骤：

1. 进入应用市场。
2. 导入模板。
3. 填写表单字段。
4. 运行一次。
5. 检查输出或下载链接。

通过标准：

-   模板导入后节点和连线完整。
-   所有表单变量使用 `{{ $form.xxx }}` 且运行时不为空。
-   LLM 输出符合模板预期。

### S04 文档 / 表格 / PPT 导出

目标：确认导出节点生成文件并返回可下载链接。

步骤：

1. 用 `DocumentExport` 导出 Markdown/JSON 为 `.docx` 或 `.md`。
2. 用 `SpreadsheetExport` 导出 JSON 数组为 `.xlsx`。
3. 用 `PptxExport` 导出结构化 slides 为 `.pptx`。
4. 下载文件并打开检查内容。

通过标准：

-   空内容不生成文件，并返回中文提示。
-   正常内容生成下载链接。
-   文件名安全，格式正确，内容可打开。

### S05 计费手动开通与硬性拦截

目标：确认商业化 V1 的计费口径可用：后台手动开通 + token/bot/seat 硬拦截。

步骤：

1. 运营账号进入 `/billing`。
2. 创建或选择套餐。
3. 给测试组织开通订阅，设置较低 token/bot/seat 额度。
4. 普通用户在 `/account` 查看额度。
5. 创建超过 bot 限额的工作流，期望 402。
6. 邀请超过 seat 限额的成员，期望 402。
7. 运行模型直到 token 达上限后，再发起下一次运行，期望 402。

通过标准：

-   `/account` 明确显示“后台手动开通，不开放自助购买”。
-   `/billing` 明确显示“V1 手动开通，不开放自助支付”。
-   三类超额均返回 HTTP 402，错误码分别为 `BOT_LIMIT_EXCEEDED`、`SEAT_LIMIT_EXCEEDED`、`TOKEN_LIMIT_EXCEEDED`。
-   运营调整套餐或覆盖额度后立即生效。

### S06 工单链路

目标：确认普通用户提单、运营回复、用户查看回复可闭环。

步骤：

1. 普通用户进入支持工单页面，新建工单。
2. 工单运营账号查看工单列表。
3. 运营回复并修改状态/优先级。
4. 普通用户刷新页面查看回复。

通过标准：

-   普通用户只能看到自己的组织和自己提交的工单。
-   运营白名单账号能看到队列并回复。
-   工单状态、优先级、最后回复时间更新正确。

### S07 国产模型 Token 计量

目标：确认国产模型返回的 usage 字段能被正确记录为计费用量。

步骤：

1. 使用至少一个国产模型运行一次 Agent。
2. 保存服务端日志中的 provider 原始 usage metadata。
3. 查询本月 `BillingUsage` 记录。
4. 对比 input/output/total token 是否进入账本。

通过标准：

-   本次运行有 dedupe key，重复记录不会加倍。
-   usage metadata 能映射到 total token。
-   若供应商不返回 token，记录为 `BLOCKED` 并登记供应商响应样例。

## 4. 外部服务 Smoke

### S08 国产云向量库真实实例

四家均需在真实测试实例上执行。

| 厂商       | 必测能力                                       | 证据                      |
| ---------- | ---------------------------------------------- | ------------------------- |
| 腾讯云     | loadMethods、auto create、upsert/search/delete | 控制台截图 + 接口日志     |
| 阿里云     | loadMethods、auto create、upsert/search/delete | 控制台截图 + 接口日志     |
| 百度智能云 | loadMethods、auto create、upsert/search/delete | 控制台截图 + SDK/接口日志 |
| 火山引擎   | loadMethods、auto create、upsert/search/delete | 控制台截图 + OpenAPI 日志 |

通过标准：

-   自动建库/集合开启时能创建缺失资源。
-   关闭自动建库/集合时，缺失资源返回中文错误。
-   检索结果能还原 LangChain Document，metadata 和 chatId 隔离正确。
-   按 ID 删除和 Record Manager 清理均生效。

### S09 支付沙箱回调

商业化 V1 不开放自助支付，但如果准备开放支付，必须先完成此项。

步骤：

1. 配置公网 HTTPS `PAYMENT_NOTIFY_BASE_URL`。
2. 配置支付宝沙箱 app id、私钥、公钥和网关。
3. 创建支付宝沙箱订单并扫码支付。
4. 确认异步回调验签成功，订单变为 `paid`，订阅激活。
5. 配置微信支付 v3 沙箱/测试商户参数。
6. 创建微信 Native 订单并模拟或完成回调。
7. 篡改金额或签名，确认回调被拒绝。

通过标准：

-   正确回调验签成功，金额按整数分校验。
-   重复 paid 回调幂等。
-   错误签名、错误金额、错误订单号均被拒绝。
-   服务端保留订单号、第三方交易号、原始回调摘要和错误日志。

## 5. 证据记录模板

每条 smoke 都必须按以下字段登记：

| 字段     | 内容                                            |
| -------- | ----------------------------------------------- |
| Case ID  | 例如 `S05`                                      |
| 日期     | `YYYY-MM-DD HH:mm:ss`                           |
| 执行人   | 姓名或邮箱                                      |
| 环境     | URL、部署方式、数据库                           |
| Commit   | release commit                                  |
| 账号     | 登录账号、组织、工作区                          |
| 输入数据 | 表单字段、上传文件、请求样例                    |
| 结果     | `PASS` / `FAIL` / `BLOCKED`                     |
| 证据     | 截图路径、日志路径、订单号、云资源 ID、导出文件 |
| 备注     | 失败原因、阻塞条件、后续 owner                  |

## 6. 上线判定

-   封闭内测：S01-S07 必须 `PASS`；S08/S09 可在没有凭证或不开放支付时记为 `BLOCKED`，但必须写明原因。
-   私有化交付：客户实际使用的模型、向量库、文件存储、数据库必须 `PASS`。
-   正式公网商业化：S01-S09 必须 `PASS`，并完成合规、支付商户、ICP 备案、监控告警和备份恢复演练。
