# P5 License Enforcement L0 Inventory

本文件是 P5 L0 只读盘点。结论先行：现有 `LicenseService` 的验签、机器指纹、导入/读取和 active -> grace -> expired 时间梯度可直接复用；`EntitlementService` 已有 tier -> seats/credits/concurrency/features 模板、license/subscription/local 三类来源和 credits 用量表，但 IAM feature gate、license valid、座位、只读/过期拦截还没有接闸。

## 8. L0 结论

-   复用优先：`LicenseService` 的 Ed25519 验签、机器指纹、导入/读取、时间梯度和 `EntitlementService` 的 tier 模板/用量扣减/limit check 都不应重写。
-   主要缺口：App 级 license cache、IAM feature gate 真校验、seat 接线、readOnly/expired 全局中间件、production key 与签发工具。
-   产品决策待确认：free seats 最终是 1 还是 2；perpetual payload 用 `model` 还是其他字段表达；IAM `feat:*` 与 entitlement features 的最终映射表。
-   本轮未修改运行代码，后续应从 L1 开始分阶段接线并逐阶段过门禁。
