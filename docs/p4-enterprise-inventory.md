## Scope

This is the Phase A read-only inventory for physically removing FlowiseAI enterprise source from the server tree. The inventory covers:

-   `packages/server/src/enterprise/**/*.ts`: 111 TypeScript files.
-   `packages/server/src/IdentityManager.ts`: 1 legacy identity boundary file.

No business code was changed in Phase A. This document is the only intended artifact before human review.

Clean-room boundary:

-   The inventory classifies capabilities and import consumers. It must not be used as permission to copy or adapt enterprise implementation code.
-   "Keep" below means keep the function in self IAM through FlowOps-owned code, not keep enterprise source.
-   Anything marked as a gap must be implemented clean-room in `iam/self/**` or another self-owned module before deleting the enterprise file that currently supplies it.

## Legend

| Class   | Meaning                                                                                                                                 |
| ------- | --------------------------------------------------------------------------------------------------------------------------------------- |
| 留-功能 | Self IAM still needs this capability or contract. Use self-owned implementation or add one clean-room before removing enterprise.       |
| 已替代  | Self IAM already has an implementation. Later phases only need to cut imports, bridges, tests, or migration indexes to self-owned code. |
| 弃      | Self IAM does not need this enterprise capability. Delete after direct consumers are removed or rewritten.                              |

Consumer notation:

-   "External" means a direct consumer outside `packages/server/src/enterprise/**`.
-   "Enterprise internal" means only enterprise files consume it; those chains disappear with the enterprise directory.
-   Tests are listed because Phase E must also stop test-time imports and mocks from pointing at removed files.

Key gaps to review before Phase B:

-   `iam/security.ts` has no self implementation yet for `getHash` and `validatePasswordOrThrow`; `commands/user.ts` still consumes those exports.
-   `LoggedInUser` and the error surface are still type-exported from `enterprise/Interface.Enterprise.ts`; self has `FlowOpsLoggedInUser` but IAM needs a self-owned public type.
-   `OrganizationUser` has no one-to-one self entity. Current self model represents membership with `FlowOpsWorkspaceMember`; `services/billing/index.ts` still counts `OrganizationUser` directly.
-   `WorkspaceShared` is not registered in self mode, but `services/credentials`, `services/openai-assistants-vector-store`, and `routes/oauth2` still import it via `iam/entities.ts`.
-   `iam/services.ts` currently contains self behavior but still exposes enterprise fallback classes; Phase B/C should replace this with real self-owned services or caller-level self branches, not no-op stubs.
-   Tests that mock enterprise files must be rewritten before Phase E: `ScheduleExecutor.test.ts`, `services/chatflows/index.test.ts`, `iam/boot.test.ts`, and `routes/iamSelfFeatureGate.test.ts`.
